# Heater Calibration Sample (포트폴리오)

## 목적
이 저장소는 히터 캘리브레이션 소프트웨어 개발 경험을  
**포트폴리오 용도로 정리한 샘플 저장소**입니다.

실제 회사에서 사용된 코드나 상용/사내 구현물은 포함되어 있지 않으며,  
개발 시 제가 어떤 구조와 방식으로 코드를 작성하는지를 보여주기 위한 목적입니다.

---

## 이 저장소에서 확인할 수 있는 내용
- 시리얼 기반 장비 통신 구조
- 통신 채널과 프로토콜 처리 분리 방식
- 테스트 흐름 제어 및 판정 로직 설계
- 로그 기록 및 결과 데이터 관리 방식

---

## 의도적으로 제외한 내용
- 회사 또는 특정 프로젝트에 종속된 구현
- UI 디자이너 자동 생성 코드
- 환경 의존적인 설정 값 (DB, FTP, 경로 등)
- 실제 장비 프로토콜 전문

---

## 전체 시스템 구조 (개념 설명)
실제 히터 캘리브레이션 소프트웨어는 다음과 같은 계층 구조로 설계되었습니다.

### 1. 통신 계층 (Communication Layer)
- 시리얼 기반 장비 통신
- 채널 추상화를 통한 I/O 로직 분리
- 프로토콜 파싱 로직을 통신 처리와 분리

### 2. 애플리케이션 계층 (Application Layer)
- 테스트 시퀀스를 제어하는 중앙 흐름 관리
- PASS / FAIL 판정 로직을 UI와 분리하여 관리

### 3. 유틸리티 계층 (Utility Layer)
- 테스트 로그 기록
- CSV 형태의 결과 데이터 저장

전체 구조에 대한 개념 설명은 `/docs/architecture.md` 파일에 정리되어 있습니다.

## 참고
이 저장소는 **실제 동작 가능한 완성 프로그램을 제공하는 것이 목적이 아니라**,  
개발자의 설계 방식과 코드 스타일을 참고하기 위한 샘플입니다.
